# Skills Lifecycle

## Current Scope

Mission Control currently supports the customer-specific Skills path through three stages:

| Stage | Owner | Status Field | Input | Output |
|---|---|---|---|---|
| Draft | 大管家 / admin | `Draft` | P4 machine-readable blueprint from `GET /api/onboarding/customer/blueprint` | Candidate Skill cards with order, reason, workflow stage, inputs, outputs, handoff, and human confirmation |
| Tenant | admin | `Tenant` | Approved P4 candidate Skills | `phase0/tenants/<tenant>/vault/skills/*.md` generated by `POST /api/onboarding/customer/skills` |
| UAT | customer + admin | `UAT` | Tenant Skill files plus P21 UAT tasks / submissions | Customer feedback in `uat-tasks.jsonl` and `uat-submissions.jsonl` |

P9 lives under Customer Onboarding at `/onboarding/customer/skills?role=admin&tenant=<tenant>`. It is the only place to configure and inject customer-specific Skills during the current phase.

`/skills` is no longer a customer Skill configuration surface. It is a Phase 2 ClawHub Marketplace planning view with a read-only cross-tenant inventory of already injected Tenant Skills.

## Stage Details

### Draft: P4 Blueprint

- Source: `vault/intake-analysis.md` machine-readable JSON.
- API: `GET /api/onboarding/customer/blueprint?tenant_id=<tenant>`.
- Required fields per candidate Skill:
  - `title` and `order`
  - `reason`
  - `workflow_stage`
  - `inputs`
  - `outputs`
  - `handoff`
  - `human_confirmation`
- Responsibility: 大管家 / admin verifies that the Skill is customer-specific and not generic.

### Tenant: P9 Vault Injection

- Action: click `Draft 注入到 Tenant Vault` on the P9 Customer Onboarding page.
- API: `POST /api/onboarding/customer/skills`.
- Output directory: `phase0/tenants/<tenant>/vault/skills/`.
- Generated path shape: `vault/skills/<skill-name>.md`.
- Existing different files are protected and reported as skipped; P9 does not overwrite customer memory silently.

### UAT: P21 Feedback

- Trigger: P21 customer UAT tasks or submissions exist for the tenant.
- Files:
  - `phase0/tenants/<tenant>/uat-tasks.jsonl`
  - `phase0/tenants/<tenant>/uat-submissions.jsonl`
- Responsibility: customer submits feedback; admin reviews and decides whether the P4/P9 blueprint needs another iteration.
- P9 status shows `UAT 验证中（P21 反馈中）` when active UAT feedback exists.

## Phase 2 Planning

The following stages are deliberately not implemented in Phase 1:

| Planned Stage | Purpose | Open Product Decisions |
|---|---|---|
| Publish Candidate | Mark a Tenant Skill as ready to leave a single customer vault | publisher identity, consent, anonymization rules |
| Review | Check safety, quality, pricing, and version metadata | reviewer role, review checklist, rejection flow |
| ClawHub Marketplace | List approved Skills for broader reuse | upload, pricing, versioning, one-click publish, install provenance |

Until Phase 2 is designed, `/skills` must stay read-only for customer Skills and must direct configuration work back to Customer Onboarding P9.
